1. Motivation

What is your dataset?

Taxi trips dataset for New York was used for the analysis. The dataset is from the website NYC.gov. The dataset contains taxi trips from yellow and green taxis but only the data for the yellow taxis were used. The dataset contains data of over 10 years, 2009-2017 but it was decided to analyse all the data from 2017. The dataset contains among other things: pickup / drop-off locations and timestamps, payment types, number of passengers and trip distance.

Why did you choose this/these particular dataset(s)?

This dataset is really popular and has been used in many data analysis. The reason for that and a part of the reasons why we chose this dataset, is because it has a lot of data, both in terms of features and in terms of a number of rows. So with many features and detailed meta-data for the trips, it is possible to investigate and analyse many different aspects of the dataset. Therefore we picked this dataset even many people have used it before and we tried to find a new angle which others might not have covered before.

What was your goal for the end user’s experience?

The goal is to tell the user a story, a story about how taxi drivers in NYC can use information from the dataset as a leverage e.g. to earn more money. We want to show the user visualisation based on big data, which can e.g. be used to:

  • decide when during the day it is best to drive based on where the taxi driver is located (or where the taxi driver lives)
  • decide where is the best pickup places for the taxi driver to maximize the length of the trip and therefore earn more money
  • decide which pickup places are better than others based on if it is a weekend or not
  • etc.
Therefore the goal is to have the story informative as well as to have the users, experience that they have learnt something useful and can use the visualisation to take valuable actions if they are or intent to be NYC taxi drivers.

2. Basic stats. Let’s understand the dataset better

Write about your choices in data cleaning and preprocessing

The dataset for all the yellow trips 2017, is around 10GB. For this big dataset, it doesn't work to have the data locally on a web server, so it has to be preprocessed and cleaned.

First, the 10GB data was downloaded on our local computers. Then all the data was uploaded to Google Cloud Storage within the Google Cloud Platform. When we had all the data in the Google Cloud we used Google BigQuery to do the data cleaning and preprocessing.

Then, different database tables for the trips data were merged, as the data has to be downloaded per month. Next unnecessary features were removed. Thereafter outliers were detected and necessary actions were taken based on the nature of the outliers. As an example, it can be detected that there has been some input error on behalf of the taxi drivers when the record number of participants. That can be seen as few datapoints (trips) have 6, 7, 8 or 9 number of participants even the yellow taxi (yellow taxi van) can maximum take 5 participants. The datapoints that were clearly affacted by input errors, were therefore removed.
